# コミュニケーション
- 日本語で応答する（コード・変数名は英語）
- 簡潔に回答し、自明な説明は省略する
- 複雑なタスクでは実装前に計画を提示し、承認後に着手する

# 開発ドキュメント
- docs/spec.mdに記載されている内容をもとに開発を進める
- ユーザーからの指示により、仕様変更が発生する場合、spec.mdの内容を適宜修正する

# Git規約
- Conventional Commits形式、本文は日本語（例: `feat: ユーザー認証にOAuth2を追加`）
- 確認なしに自動コミット・自動pushしない

# 実装方針
- Reactコンポーネントを実装するときは、親要素に `data-component`、小要素に `data-element` 属性を付与する

# 禁止事項
- README・ドキュメントを勝手に生成・変更しない
- 既存の動作するコードを理由なくリファクタリングしない
- ユーザーのプロンプトが疑問形で終わる場合・質問であると判断できる場合、確認なしにファイルを変更しない。
